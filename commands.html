Command List

blargbot is equipped with a number of miscellaneous commands. By default, blargbot uses the command prefix '!'. This can be changed with the setprefix command. It will also respond to it's name, 'blargbot'.

Some commands have permission requirements. In order to satisfy them, you need to have a role with the name detailed in the Permissions section.

General commands

cah

Usage: cah
Permissions: none

Generates a random set of Cards Against Humanity cards.

cat

Usage: cat
Permissions: none

Displays a picture of a cat, taken from random.cat

danbooru

Usage: danbooru <tags>
Permissions: 'Porn Summoner' (if not safe)

Displays three images obtained from danbooru.donmai.us using the provided tags. You can use up to 2 tags at a time. Results have the possibility of being NSFW. If the current channel is not designated as NSFW, a user needs the role by the name of "Porn Summoner" in order to use this command without including the 'rating:safe' tag.

e621

Usage: e621 <tags>
Permissions: 'Porn Summoner' (if not safe)

Displays three images obtained from e621.net using the provided tags. You can use up to 6 tags at a time. Results have the possibility of being NSFW. If the current channel is not designated as NSFW, a user needs the role by the name of "Porn Summoner" in order to use this command without including the 'rating:safe' tag.

econ

Usage: econ <from> <to> <amount>
Permissions: none

Converts currency using recent rates. Example:

User> blargbot econ USD CAD 1
blargbot> @User, 1.0 USD is equivalent to X.X CAD

Currency codes are not case sensitive.

free

Usage: free <caption> [ | lowercaption]
Permissions: none

Tells everyone what you got for free.

tag

Usage: tag [<name> | create | edit | delete | raw | author | search | list | help]
Permissions: none

Tags are like public custom commands. You can create them on one guild, and use them on another. Anyone is capable of making tags. Tags use a tagging system, so they can can range from simple to complex. See the tag page for more details. 

tag <name>

Executes a tag with the given name. 

tag create <name> <content>

Creates a tag with the given name and content. The name must be unique. 

tag edit <name> <content>

Edits an existing tag. You must own it to edit it!

tag delete <name>

Deletes the specified tag. You must own it to delete it!

tag raw <name>

Outputs the raw tag

tag author <name>

Tells you who made the specified tag

tag search <name>

Searches for tags with given name

tag list

Lists all tags

tag help

Gets basic tag help.

time

Usage: time <timezone> [ <timezone2> <time> ]
Permissions: none

Tells you the current time in the specified timezone. If timezone2 and time are specified, converts the time from timezone to timezone2. Time must be formatted as hh:mm[AM/PM], and timezones must use these timezone codes.

Bot Commander Commands

blacklist

Usage: blacklist
Permissions: 'Bot Commander'

Prevents the bot from speaking in the current channel. Execute the command again to disable.

ccommand

Usage: ccommand <command name> [command content]
Permissions: 'Bot Commander'

Creates a simple custom command. Once created, whenever you say the command name, blargbot will respond with the command content. For example:

User> blargbot ccommand test This is a test command.
blargbot> Created command test
User> blargbot test
blargbot> This is a test command.

You can also use the tagging system to make more powerful commands.

input: Hello, {username}.
output: Hello, User

To remove a custom command, simply do ccommand without command content. For example:

User> blargbot ccommand test
blargbot> Removed command test
User> blargbot test
*no response from blargbot*

Finally, custom commands take precedent over all other commands. As such, you can use it to overwrite commands, or disable them entirely. If the command content is "null" (without the quotations), blargbot will have no output whatsoever, allowing you to disable any built-in command you wish. You cannot overwrite the 'ccommand' command.

farewell

Usage: farewell [farewell message]
Permissions: 'Bot Commander'

Sets a farewell message for any user who leaves your guild. You can use the tagging system for more customization. For example:

User> blargbot farewell **{username}** has left. Bye!
blargbot> Greeting set. Simulation: **User has left. Bye!

greet

Usage: greet [greeting]
Permissions: 'Bot Commander'

Sets a greeting for any new user who joins your guild. You can use the tagging system for more customization. For example:

User> blargbot greet Welcome, **{username}**. Please read #rules.
blargbot> Greeting set. Simulation: Welcome, **User**. Please read #rules.

nsfw

Usage: nsfw
Permissions: 'Bot Commander'

Designates the current channel as NSFW, allowing any user to do a NSFW command with the 'Porn Summoner' role.

setprefix

Usage: setprefix <prefix>
Permissions: 'Bot Commander'

Sets the command prefix for the guild. You can set it to anything, as long as it's not blank. Set it to 'null' to disable it entirely and use only 'blargbot' as a prefix.

Admin Commands

ban

Usage: ban <user> [days]
Permissions: 'Admin'

Bans a user from the guild. Days specifies how many days of messages to delete.

If mod-logging is enabled, the ban will be logged.

kick

Usage: kick <user>
Permissions: 'Admin'

Kicks a user from the guild.

If mod-logging is enabled, the kick will be logged.

ban

Usage: modlog [disable]
Permissions: 'Admin'

Enables the modlog and sets it to the current channel. Doing modlog disable will disable it. When an admin does a moderation command (ban, unban, mute, unmute, and kick), the incident will be logged. The admin will then be encouraged to do reason <case number> <reason> to specify why the action took place.

Bans and unbans are logged regardless of whether the ban or unban commands are used.

mute

Usage: mute <user>
Permissions: 'Admin'

Gives the user a special muted role. On first run, this role will be created. The bot needs to be able to manage roles to create and assign the role, and manage channels to configure the role. You are able to manually configure the role without the bot, but the bot has to make it. Deleting the muted role causes it to be regenerated.

If mod-logging is enabled, the mute will be logged.

purge

Usage: purge
Permissions: 'Admin'

Deletes all the messages in the current channel made by the bot, within 100 messages.

reason

Usage: reason <case number> <reason>
Permissions: 'Admin'

Sets the reason for an action on the modlog.

tidy

Usage: tidy [amount]
Permissions: 'Admin'

Cleans a number of messages, defaulting to 25.

unban

Usage: unban <user>
Permissions: 'Admin'

Unbans a user.

If mod-logging is enabled, the unban will be logged.

unmute

Usage: unmute <user>
Permissions: 'Admin'

Unmutes a user.

If mod-logging is enabled, the unmute will be logged.
